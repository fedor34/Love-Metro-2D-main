# Исправление v2: VIP персонажи проходят через границы поля

## История проблемы
После первого исправления (учет pivot точки в расчете offset) ошибка появилась СНОВА. Проведено повторное расследование.

## РЕАЛЬНАЯ ПРИЧИНА ОШИБКИ

### Проблема №1 (первое исправление - НЕВЕРНОЕ):
Предполагалось, что проблема в неправильном расчете offset коллайдера из-за разных pivot точек спрайтов. **ЭТО ОКАЗАЛОСЬ НЕВЕРНЫМ**.

### Проблема №2 (настоящая причина - НАЙДЕНА):
**Функция `NormalizeVipCollider()` ПЕРЕЗАПИСЫВАЛА правильные коллайдеры из префабов!**

#### Детали:
1. **VIP префабы УЖЕ ИМЕЮТ правильные коллайдеры:**
   ```yaml
   PassangerMale_VIP.prefab и PassangerFemale_VIP.prefab:
   m_Offset: {x: 0, y: 0.17477119}
   m_Size: {x: 1.6904762, y: 0.5632155}
   ```
   Эти настройки ИДЕНТИЧНЫ обычным персонажам и работают корректно!

2. **Функция `NormalizeVipCollider()` вызывалась в `Awake()`:**
   ```csharp
   // СТАРЫЙ КОД (неправильный):
   if (ShouldNormalizeVipCollider())
   {
       NormalizeVipCollider(); // <- ПЕРЕЗАПИСЫВАЕТ правильный коллайдер!
   }
   ```

3. **Функция пересчитывала offset, основываясь на pivot:**
   - Для VIP Male с pivot в центре: offset становился ≈ **-1.85**
   - Исходный правильный offset был: **+0.17**
   - **Разница ~2 единицы!** Коллайдер сдвигался слишком низко

4. **Результат:**
   - Коллайдер VIP персонажей оказывался значительно ниже, чем должен
   - Верхняя часть спрайта оставалась без коллайдера
   - При движении вверх персонажи проходили через границы

## РЕШЕНИЕ

Изменена логика `NormalizeVipCollider()`:

```csharp
// НОВЫЙ КОД (правильный):
private void NormalizeVipCollider()
{
    var box = GetComponent<BoxCollider2D>();
    // ... проверки ...
    
    // КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ: НЕ ПЕРЕЗАПИСЫВАЕМ правильные коллайдеры!
    if (box.size.magnitude > 0.1f)
    {
        // Коллайдер уже настроен - оставляем как есть
        Diagnostics.Log($"[Passenger][vip-collider] {name}: Collider already configured correctly, skipping normalization");
        box.isTrigger = false;  // только гарантируем, что не триггер
        box.usedByEffector = false;
        return;  // <- НЕ ТРОГАЕМ коллайдер!
    }
    
    // Только если коллайдер пустой - создаем новый
    // ... (код создания коллайдера для случаев, когда префаб без коллайдера)
}
```

### Ключевое отличие:
- **Раньше:** функция ВСЕГДА пересчитывала коллайдер для VIP
- **Теперь:** функция СОХРАНЯЕТ существующие коллайдеры, только проверяет флаги

## Дополнительные находки расследования:

1. **Layer настройки:**
   - Все персонажи (VIP и обычные) на слое `Default` (0) ✓
   - Layer правильно устанавливается в `Initiate()` ✓

2. **Rigidbody2D настройки:**
   - IncludeLayers: 0 (все слои) ✓
   - ExcludeLayers: 0 (нет исключений) ✓
   - CollisionDetection устанавливается в Continuous в Awake() ✓

3. **Physics2D Layer Collision Matrix:**
   - Default <-> Default: ✓ ENABLED
   - Default <-> Wall: ✓ ENABLED
   - Default <-> SoftWall: ✓ ENABLED
