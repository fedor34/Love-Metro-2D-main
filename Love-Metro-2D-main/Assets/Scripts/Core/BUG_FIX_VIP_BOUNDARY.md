# Исправление: VIP персонажи проходят через границы поля

## Описание проблемы
VIP персонажи могли свободно проходить через границы поля, в то время как обычные персонажи корректно блокировались. Это происходило постоянно и нарушало игровой процесс.

## Причина ошибки
Проблема была в методе `NormalizeVipCollider()` в файле `Passenger.cs` (строки 1102-1141).

### Техническое объяснение:
1. **Разные pivot точки спрайтов:**
   - VIP Male спрайт имеет pivot в центре: `{x: 0.5, y: 0.5}`
   - Обычные спрайты имеют pivot внизу: `{x: 0.5, y: 0}`

2. **Неправильный расчет offset:**
   ```csharp
   // СТАРЫЙ КОД (неправильный):
   float offsetY = (-s.y * 0.5f) + (height * 0.5f) + footMargin;
   ```
   Этот расчет НЕ учитывал pivot точку спрайта, что приводило к тому, что:
   - Для VIP Male с pivot в центре: offset получался около **-1.85**
   - Для обычных с pivot внизу: offset был около **+0.17**

3. **Результат:**
   - Коллайдер VIP персонажей сдвигался слишком низко (на ~2 единицы вниз от центра GameObject)
   - Верхняя часть спрайта оставалась БЕЗ КОЛЛАЙДЕРА
   - При движении вверх физический движок не обнаруживал столкновения с границами
   - Персонаж проходил через границу, так как его коллайдер был ниже

## Решение
Исправлен расчет `offsetY` с учетом нормализованной pivot точки спрайта:

```csharp
// НОВЫЙ КОД (правильный):
Vector2 pivotNormalized = new Vector2(
    pivot.x / sprite.rect.width,
    pivot.y / sprite.rect.height
);

float spriteBottomInLocal = -s.y * pivotNormalized.y;
float offsetY = spriteBottomInLocal + (height * 0.5f) + footMargin;
```

Теперь коллайдер правильно размещается у основания ног персонажа независимо от pivot точки спрайта.
